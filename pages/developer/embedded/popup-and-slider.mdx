## Create a popup

1. Go to the <span className="text-cyan-500 font-bold bg-cyan-100 px-2 py-1 rounded">Share</span> tab of your published form and select <span className="text-cyan-500 font-bold bg-cyan-100 px-2 py-1 rounded">Popup</span>
2. Configure your popup options
3. Click <span className="text-cyan-500 font-bold bg-cyan-100 px-2 py-1 rounded">Get the code</span> to copy the embed snippet
4. Paste the code into your website

### Button trigger

The simplest way to open a popup — users click a button:

```html copy
<button
  data-makeform-popup="https://makeform.ai/e/YOUR_FORM_ID"
  data-makeform-width="500"
  data-makeform-events
>
  Open Form
</button>

```

## Popup & slider options

### Width

Set the modal width in pixels using the `data-makeform-width` attribute. Values are clamped between **360px** and **800px**. Default is `500`.

On mobile (viewport ≤ 640px), popups and sliders automatically expand to full screen regardless of the width setting.

### Background color

Use `data-makeform-bg` to set an initial background color for the modal. This prevents a white flash while the form loads.

```html
<button
  data-makeform-popup="https://makeform.ai/e/YOUR_FORM_ID"
  data-makeform-bg="#f0f9ff"
>
  Open Form
</button>
```

### Auto-close after submit

Automatically close the popup or slider after a successful form submission. Set the delay in seconds (0–30) using `data-makeform-auto-close`:

```html
<button
  data-makeform-popup="https://makeform.ai/e/YOUR_FORM_ID"
  data-makeform-auto-close="3"
>
  Open Form
</button>
```

Set to `0` for no auto-close (user must close manually).

## Auto-open triggers

Instead of requiring a button click, you can configure popups and sliders to open automatically based on user behavior.

<div className="shadow-2xl mt-12">![Auto-open trigger options](/images/popup_slider3.png)</div>

Use `window.MakeFormConfig` to configure auto-open:

### Page load

Open the form immediately when the page loads:

```html copy
<script>
window.MakeFormConfig = {
  formId: "YOUR_FORM_ID",
  popup: {
    width: 500,
    bgColor: "#ffffff"
  }
};
</script>
<script src="https://www.makeform.ai/widgets/embed.js"></script>
```

When no `open` trigger is specified, the form opens on page load by default.

### Time delay

Open the form after a specified delay:

```html copy
<script>
window.MakeFormConfig = {
  formId: "YOUR_FORM_ID",
  popup: {
    width: 500,
    open: {
      trigger: "time",
      ms: 5000  // 5 seconds (500–120000)
    }
  }
};
</script>
<script src="https://www.makeform.ai/widgets/embed.js"></script>
```

### Exit intent

Open the form when the user moves their cursor toward the top of the browser window (suggesting they're about to leave):

```html copy
<script>
window.MakeFormConfig = {
  formId: "YOUR_FORM_ID",
  popup: {
    width: 500,
    open: {
      trigger: "exit"
    }
  }
};
</script>
<script src="https://www.makeform.ai/widgets/embed.js"></script>
```

<Callout type="info">
Exit intent detection is not supported on mobile devices, as there is no cursor movement to track.
</Callout>

### Scroll depth

Open the form when the user scrolls past a percentage of the page:

```html copy
<script>
window.MakeFormConfig = {
  formId: "YOUR_FORM_ID",
  popup: {
    width: 500,
    open: {
      trigger: "scroll",
      scrollPercent: 50  // 1–100
    }
  }
};
</script>
<script src="https://www.makeform.ai/widgets/embed.js"></script>
```

### Using sliders with auto-open

Replace `popup` with `slider` in the configuration:

```javascript copy
window.MakeFormConfig = {
  formId: "YOUR_FORM_ID",
  slider: {
    width: 500,
    open: {
      trigger: "time",
      ms: 3000
    }
  }
};
```

## Multiple auto-open forms

Use `window.MakeFormConfigs` (array) to configure multiple auto-open forms on the same page:

```html copy
<script>
window.MakeFormConfigs = [
  {
    formId: "FORM_ID_1",
    popup: {
      width: 500,
      open: { trigger: "time", ms: 5000 }
    }
  },
  {
    formId: "FORM_ID_2",
    slider: {
      width: 400,
      open: { trigger: "scroll", scrollPercent: 75 }
    }
  }
];
</script>
<script src="https://www.makeform.ai/widgets/embed.js"></script>
```

<Callout type="info">
You can configure up to 20 auto-open forms per page.
</Callout>

## Suppression options

Control when and how often auto-open forms appear to visitors.

<div style={{ backgroundColor: '#fdf2f8', border: '2px solid #ec4899', borderRadius: '0.5rem', padding: '1rem', marginTop: '1.5rem', marginBottom: '1.5rem', display: 'flex', gap: '0.75rem', alignItems: 'flex-start' }}>
  <div style={{ fontSize: '1.5rem', lineHeight: '1' }}>✨</div>
  <div style={{ flex: '1' }}>
    <span style={{ color: '#831843', fontSize: '0.95rem' }}>
      Suppression options (`showOnce` and `doNotShowAfterSubmit`) are available with <a href="/plans-and-pricing" style={{ textDecoration: 'underline', color: '#831843' }}><strong>Makeform Pro</strong></a>.
    </span>
  </div>
</div>

### Show once per visitor

Only show the form once per visitor. After the first display, the form won't appear again:

```javascript
window.MakeFormConfig = {
  formId: "YOUR_FORM_ID",
  popup: {
    showOnce: true,
    open: { trigger: "time", ms: 3000 }
  }
};
```

This uses browser `localStorage` to track whether the form has been shown (key: `mf_shown_{formId}`).

### Don't show after submit

Hide the form permanently after the visitor has submitted it:

```javascript
window.MakeFormConfig = {
  formId: "YOUR_FORM_ID",
  popup: {
    doNotShowAfterSubmit: true,
    open: { trigger: "time", ms: 3000 }
  }
};
```

Uses `localStorage` key `mf_submitted_{formId}` to track submissions.

### Combining suppression options

You can use both options together:

```javascript
window.MakeFormConfig = {
  formId: "YOUR_FORM_ID",
  popup: {
    showOnce: true,
    doNotShowAfterSubmit: true,
    autoClose: 3000,
    open: { trigger: "scroll", scrollPercent: 50 }
  }
};
```

## Callbacks

Use the JavaScript API to react to popup and slider lifecycle events:

```javascript copy
MakeForm.openPopup("YOUR_FORM_ID", {
  width: 500,
  onOpen: ({ formId }) => {
    console.log("Popup opened:", formId);
  },
  onReady: ({ formId }) => {
    console.log("Form loaded:", formId);
  },
  onSubmit: ({ formId, data }) => {
    console.log("Form submitted:", formId, data);
    // data contains { [fieldName]: value }
  },
  onClose: ({ formId }) => {
    console.log("Popup closed:", formId);
  }
});
```

The same callbacks work with `MakeForm.openSlider()`.

See <Link href="/developer/embedded/developer-resources">Developer Resources</Link> for the complete JavaScript API reference.

## Mobile behavior

On viewports **640px or narrower**, popups and sliders automatically switch to full-screen mode:

- **Popup**: Expands to 100% width and height, no border radius
- **Slider**: Expands to 100% width, full viewport height

The width setting is overridden on mobile to ensure a good experience on small screens.
