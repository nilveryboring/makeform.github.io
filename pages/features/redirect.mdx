# Redirect After Submission

Redirect respondents to another page after they submit a form.

![Redirect after submission settings](/images/redirect-rule.webp)

<Callout type="info">
Use redirects when the next step should happen outside the form, such as booking a meeting, opening a checkout page, or sending someone to a resource.
</Callout>

## When to use a redirect

Redirects are useful when a normal thank-you page is not enough.

Common destinations include:

- A booking page for qualified leads
- A checkout page after an order form
- A download page after a lead magnet form
- A help article or onboarding guide
- A custom thank-you page on your own website

## Set up a redirect

1. Open your form in the builder.
2. Go to the form's ending or submission settings.
3. Choose <span className="text-cyan-500 font-bold bg-cyan-100 px-2 py-1 rounded">Redirect to URL</span> as the post-submission action.
4. Enter the full destination URL, including `https://`.
5. Save the form.
6. Preview the form and submit a test response.

## What happens after submission

![Redirect submission flow](/images/redirect-preview.webp)

Makeform saves the response first. Then it opens the destination URL you configured.

This means you can still review the submission in Makeform even if the respondent continues to another site.

## Good redirect patterns

| Form type | Redirect destination |
| --- | --- |
| Demo request | Booking calendar |
| Paid intake | Checkout page |
| Event registration | Confirmation page |
| Lead magnet | Download page |
| Onboarding form | Next-step guide |

## Redirect with tracking values

If your destination page needs context, combine redirects with pre-populated or hidden values.

For example, you can pass:

- `email`
- `plan`
- `source`
- `customer_id`
- `submission_id`

Only pass values that are safe to include in a URL.

## Testing checklist

- Use a full `https://` URL.
- Submit a test response.
- Confirm the response is saved in Makeform.
- Confirm the browser opens the correct destination.
- Test the redirect on mobile.
- If you pass tracking values, confirm the destination receives them.

## Troubleshooting

### The redirect does not open

Check that the destination URL starts with `https://` and is publicly reachable.

### Respondents see the wrong page

Confirm you saved the latest form settings and are testing the published version of the form.

### The response is missing

Redirects happen after the response is saved. If the response is missing, test the form without the redirect first to isolate the issue.

### Tracking values are missing

Check the destination URL and make sure query parameters are encoded correctly.
