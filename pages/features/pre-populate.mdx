# Pre-Populate Forms

Pre-populate forms by passing known values through the form URL. This lets you fill fields before the respondent starts.

![Pre-populate form URL example](/images/prepopulate-url.webp)

<Callout type="info">
Pre-population is useful for email campaigns, CRM workflows, customer portals, and any flow where you already know part of the respondent's information.
</Callout>

## How pre-population works

Add query parameters to the form URL. Makeform reads those values and fills matching fields.

Example:

```text
https://makeform.ai/f/customer-intake?email=maya@example.com&plan=pro
```

In this example:

- `email` fills the email field.
- `plan` fills the plan interest field.

## Prepare your form fields

1. Open your form in the builder.
2. Give each field you want to prefill a stable field name.
3. Use short field names such as `email`, `name`, `plan`, `company`, or `source`.
4. Publish or preview the form.
5. Add query parameters to the form URL.

## Build a pre-populated link

Start with the form URL, then add a question mark and key-value pairs:

```text
?email=maya@example.com&company=Acme&source=newsletter
```

Use `&` between each value.

If a value contains spaces or special characters, encode it:

```text
company=Acme%20Studio
```

## Respondent experience

![Pre-populated form result](/images/prepopulate-result.webp)

The respondent opens the form and sees known values already filled in. They can review and update visible fields before submitting.

## Common use cases

| Use case | Values to pass |
| --- | --- |
| Newsletter campaign | `email`, `source`, `campaign` |
| CRM lead routing | `company`, `owner`, `lead_id` |
| Customer support | `email`, `account_id`, `issue_type` |
| Event registration | `ticket_type`, `referrer`, `discount_code` |

## Use hidden fields with pre-population

If you want to pass tracking values that respondents should not edit, use <span className="text-cyan-500 font-bold bg-cyan-100 px-2 py-1 rounded">Hidden Fields</span>.

Hidden fields are good for values like:

- `utm_source`
- `utm_campaign`
- `customer_id`
- `lead_owner`
- `referrer`

## Testing checklist

- Open the link in a private browser window.
- Confirm each visible field is filled correctly.
- Confirm hidden tracking fields appear in the submission data.
- Test values with spaces and special characters.
- Submit a test response and review the saved values.

## Troubleshooting

### A field does not prefill

Check that the query parameter name matches the field name exactly.

### Spaces appear incorrectly

Encode spaces as `%20` or use a URL builder before sending the link.

### The value is visible but should be hidden

Move that value to a hidden field so respondents cannot edit it.
